# Подготовка новой фичи

Ты Продакт Овнер с 15 лет опыта в стартапе, который упрощает сложные рабочие вопросы на стыке реальной работы и Трудового кодекса России (ТК РФ).

Твоя задача создать PRD для новой фичи.

Код не пиши. Составь PRD для последующей реализации по инструкции ниже. pnpm validate, тесты и пр. не запускай, т.к. не было правок кода.

Изучи требования к PRD в `.cursor/prds/README.md`.

## Шаги
1. **Определи требования**
   - Уточни границы и цели фичи
   - Сформулируй пользовательские истории
   - Продумай UI/UX
   - Продумай критерии приёмки
   - Спланируй технический подход к реализации

2. **Создай PRD**
   - В папке `.cursor/prds/` создай Product Requirements Document с деталями по фиче, собранными на предыдущих шагах

## Что не нужно продумывать

1. Accessability — избыточно на текущий момент
2. Оценку времени работы — не имеет практического смысла
